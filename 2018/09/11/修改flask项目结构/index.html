修改flask项目结构

发表于 2018-09-11 | 分类于 web |
字数统计: 779
使用蓝图讲flask项目分离成MVC模式
  • pip install blueprint
  • 新建一个文件夹xxx，新建一个views.py文件
  • 将主py文件下的路由函数全部迁移到views.py文件下
使用manage管理app
  • pip install flask-script
  • from flask-script import Manage 导入Manage包
  • manage = Manage(app) 实例化manage
  • 将xxx.run() 改为 manage.run()
  • python xxx.py runserver -h 127.0.0.1 -p 8888 -d
请求方式及url定义
  • url_prefix = ‘/xxx’
  • @xxx.route(‘/xxx’, methods=[‘post’, ‘get’])
获取参数
  • post: request.form
  • get: request.args
建立工具类文件夹
  • 工具类文件夹格式类似于下图：
  • setting.py: 存放配置如静态文件目录等
文件存放目录
  • 在Django中一般将文件存放在media目录下，但是由于flask可以识别的静态文件目录只有templates和static，所以需要将media目录放在static文件夹之下。
  • 图片保存路径’static/media/upload/xxx.xxx’
  • 通过获取到文件之后，例如获取到文件’a’，通过a.filename可以获取到文件名，通过a.mimename可以获取到文件格式等数据，为了将文件保存到upload目录下，我们需要使用os模块
创建错误返回json数据的文件
  • 一个完整的系统有很多不同类型的返回错误，而很多错误是可以重复调用的，所以可以将所有返回信息存放在一个文件中
